# ISO 27001 — Information Security Management System

## What it is
ISO 27001 is the international standard for information security management systems (ISMS). It specifies requirements for establishing, implementing, maintaining, and continually improving an ISMS.

## Core structure
The standard uses Annex SL (high-level structure) making it compatible with ISO 9001 and ISO 14001. It has 10 clauses and Annex A with 93 controls across 4 themes.

## Annex A control themes
- Organisational controls (37 controls) — policies, roles, threat intelligence, supplier relationships
- People controls (8 controls) — screening, awareness, training, disciplinary process
- Physical controls (14 controls) — physical security perimeters, clear desk, equipment security
- Technological controls (34 controls) — access control, cryptography, logging, vulnerability management

## Key requirements for compliance
- Risk assessment and risk treatment plan documented
- Statement of Applicability (SoA) — which controls apply and why
- Information security objectives measured
- Internal audit programme
- Management review
- Nonconformity and corrective action process

## Common controls relevant to Aria
- A.8.2 Privileged access rights — review and restrict privileged accounts
- A.8.5 Secure authentication — MFA, password policies
- A.8.7 Protection against malware — EDR, endpoint protection
- A.8.8 Management of technical vulnerabilities — patching cadence, vulnerability scanning
- A.8.15 Logging — security event logs retained, protected, reviewed
- A.8.16 Monitoring activities — anomaly detection, alerting
- A.8.20 Networks security — segmentation, firewall rules
- A.8.21 Security of network services — encryption in transit, mTLS

## Certification process
Gap assessment → risk assessment → implement controls → internal audit → Stage 1 audit (document review) → Stage 2 audit (implementation review) → certification → surveillance audits annually → recertification every 3 years.

## Relationship to other standards
ISO 27001 is the management framework. ISO 27002 provides implementation guidance for Annex A controls. ISO 27005 covers information security risk management.
